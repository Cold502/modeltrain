摘  要

项目以统计局提出的统计工作“三个提高”原则为背景，以“智慧统计”的信息化建设为目标，依托运营商网络和互联网等工具，建成一套高效、科学的调查统计工作模式，能够对全区的调查统计工作进行优化和升级，节约时间成本、人力成本和资源开销；能够更加便捷的开展调查工作；同时能够实现随时随地进行填报。
使用问卷调查的方法，是采集数据或资料的最常用途径，问卷调查适用面广、方便获取数据、研究成本低，在很多领域使用非常广泛，以往由于信息化不成熟而采取传统的电话采访、纸质填写问卷等这种耗时又繁琐的方式。随着个人计算机的普及和互联网的广泛应用，各种基于网络的调查方法开始出现。如问卷星、调查派等。
在线的问卷调查系统在调查范围、调查效率、资料整理等方面具有极大的优势，结合Java Web成熟稳定的软件技术，强强联合，是未来统计业务在调查领域的主要方式。

关键词：JSP；统计；调查；问卷；JAVA；设计；开发。




一、
综述
(一)系统建设背景
国家统计局提出了统计工作“三个提高”原则――提高统计能力、提高统计数据质量、提高政府统计公信力。海淀统计局提出了“智慧统计”的信息化建设目标，以提高数据质量建设为重点，以强化统计法制建设为保障，以改革调查方法、增加调查手段，提升业务效率为基础方法，确保统计信息化系统建设做到体系完整、目标明确、发展有序、成果显著。
(二)研究意义
统计智慧调查系统项目通过对现有业务进行梳理分析，依托运营商网络和互联网等工具，建成一套高效、科学的调查统计工作模式，能够对全区的调查统计工作进行优化和升级，节约时间成本、人力成本和资源开销；能够更加便捷的开展调查工作；同时能够实现随时随地进行填报，自动实时上传的功能。在提高效率的同时系统也使得被调查对象更有意愿参与这项工作。本项目建设是在政府业务工作中应用新的、先进的方式，可以节约纸张等实物，节约人工收发的时间成本和人力成本。
(三)术语定义
系统：统计智慧调查系统。
子系统：是按照业务梳理进行模块分类的统称。


(四)技术选型
1.后端
Java是一种广泛应用于Web应用开发、移动应用开发、企业级应用开发、大数据处理等多种场景的高级编程语言。Java由Sun公司于1995年推出，现在由Oracle公司维护和发展。Java最大的特点是“一次编写，到处运行”，这意味着开发者只需要编写一次Java代码，即可在支持Java虚拟机（JVM）的任何平台上运行，无需针对每个平台单独编译。
Java语言特性包括面向对象、安全性、健壮性、高性能和可移植性。Java类库丰富，涵盖网络、数据库连接、多线程、图形用户界面开发等多个领域。
本系统是基于Java Web技术构建的，也使用了基于Java的一些Web框架，如Spring Framework、Spring MVC、MyBatis等Java EE技术栈。 
2.前端
JSP（JavaServer Pages）： JSP是一种基于Java技术的服务器端脚本语言，主要用于开发动态网页。JSP页面可以结合HTML，允许开发人员在HTML页面中嵌入Java代码和表达式，以便在服务器端处理动态内容生成。JSP文件经过JSP容器解析后转化为Servlet，然后由Servlet生成动态网页内容返回给客户端浏览器。在本项目中JSP作为构建系统的一部分，被用于呈现动态网页界面，并处理用户请求与服务器之间的交互。
系统用到其他前端技术如：JavaScript、JQuery 、Twitter Bootstrap等。
3.数据库
MySQL是一款开源的关系型数据库管理系统（RDBMS），由瑞典MySQL AB公司开发，目前隶属于Oracle公司。MySQL以其稳定、可靠、高效和易于管理的特性，在全球范围内被广泛应用，尤其适合网站、Web应用以及嵌入式应用的数据库存储解决方案。
MySQL支持标准的，可以处理大量数据，并具有良好的可伸缩性。MySQL提供的特性包括但不限于：
支持多种存储引擎，如InnoDB（支持事务处理和行级锁定）、MyISAM（快速读取）等，可以根据不同应用场景选择合适的存储引擎。
提供ACID事务支持，保证数据的一致性和可靠性。
支持多用户并发访问，有完善的权限管理和安全性控制。
支持跨平台运行，可在Windows、Linux等多种操作系统环境下使用。
具备良好的备份恢复机制，可通过多种方式进行数据备份和还原操作。
在项目中，MySQL作为系统的数据库平台，承载了系统中所有的数据存储和查询需求，为系统的高效稳定运行提供了有力的数据支持。
4.容器
Tomcat容器是一种开源的轻量级应用服务器，它主要用于部署和运行基于Java Servlet和JavaServer Pages (JSP) 技术构建的Web应用程序。Tomcat是由Apache软件基金会管理的项目，是Java Servlet和JSP规范的官方参考实现，也就是说，它是Java Web开发中一种标准的Servlet容器。
在Web应用程序部署到Tomcat容器后，容器负责监听HTTP请求，根据请求的URL路由规则，将请求分配给相应的Servlet进行处理。Servlet可以生成动态内容，并通过Tomcat容器将响应返回给客户端浏览器。

二、需求分析
(一)系统业务总体需求
从满足调查统计业务工作职能的角度出发，项目将建设一个满足统计局常规调查的业务系统，其中重点解决常规调查中各科室调查对象与指标不一致、工作量巨大、数据分散、核对困难、历史数据查找困难等问题。
根据对系统应用的调研以及各个部门的需求分析，通过高内聚、松耦合、模块复用的原则，我们为系统细化为以下几个大模块：
序号	功能模块	功能描述
1	问卷管理	主要是能够对问卷的模板进行设计，能够对问卷进行权限的控制与发布。
2	项目管理	系统对项目进行管理与维护，用户可以在自己权限范围内进行项目的添加与维护。项目数据的维护是问卷等数据创建的基础。
3	样本管理	对问卷调查的调查对象即样本进行分类管理与维护。
4	调查员管理	对调查员信息管理以及调查员任务管理
5	填报过程管理	对任务开始调查到结束的时间段内填报情况的管理
6	基础数据管理	系统运行的基础，包括对系统的基础数据进行管理，包括对系统菜单的设置，包括对系统用户权限的设置。

5.关键程序代码如下
/**
 * 项目分析
 * @param projectMain
 * @param request
 * @param response
 * @param model
 * @return
 */
@RequiresPermissions("project:projectMain:view")
@RequestMapping(value = "analysis")
public String analysis(ProjectMain projectMain, HttpServletRequest request, HttpServletResponse response,
      Model model) {
/** 取抽样用户ID */
   String simpleUserId = request.getParameter("simpleUserId");
   String projectId = projectMain.getId();
   model.addAttribute("projectMain", projectMain);

   /** 根据项目查询问卷 */
   List<String> quesIds = projectAnalysisService.findIds(projectId);
   Map<String, String> map = new HashMap<String, String>();
   map.put("quesId", quesIds.get(0));
   map.put("projectId", projectId);
   map.put("simpleUserId", simpleUserId);
   if ("3".equals(projectMain.getReportingMethods())) {
      List simpleInfos = projectAnalysisService.getSample(projectMain);
      model.addAttribute("simpleInfos", simpleInfos);
   }
   /** 柱状图形数据 */
   List<String> jd = projectAnalysisService.getjd(map);
   List<Integer> list = new ArrayList<Integer>();
   list.add(Integer.parseInt(jd.get(0)));

   /** 当项目填报类型为匿名时,未完成为0 */
   if ("2".equals(projectMain.getReportingMethods()) 
         || "3".equals(projectMain.getReportingMethods())
         || "4".equals(projectMain.getReportingMethods())) {
      list.add(0);
   } else {
int v = Integer.parseInt(jd.get(1)) - Integer.parseInt(jd.get(0));
      list.add(v);
   }
   String data = JacksonUtils.converBean2Json(list);
   /** 线性图形数据 */
   List jdline = projectAnalysisService.getjdline(map);
   List<String> time = new ArrayList<String>();
   List<Integer> count = new ArrayList<Integer>();
   for (Map line : jdline) {
      time.add(String.valueOf(line.get("time")));
      count.add(Integer.parseInt(line.get("count").toString()));
   }
/** json转换 */
   String times = JacksonUtils.converBean2Json(time);
   String counts = JacksonUtils.converBean2Json(count);
/** 设置页面返回值 */
   request.setAttribute("simpleUserId", simpleUserId);
   request.setAttribute("times", times);
   request.setAttribute("counts", counts);
   request.setAttribute("maxNum", Integer.parseInt(jd.get(1)));
   request.setAttribute("data", data);
   request.setAttribute("projectMain", projectMain);
   /** 返回页面 */
   return "hdzxdctj/project/project_analysis";
}
该核心程序代码主要实现了对项目中的问卷进行分析的功能，查询出进度、频次等数据，并设置柱状图和线性图的数据。在成功操作后向用户反馈相应的结果信息后，引导用户跳转至分析展示页面。
